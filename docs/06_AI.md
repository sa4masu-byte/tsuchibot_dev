# 1. AI Objective

AI is used where interpretation is valuable and deterministic rules are insufficient.

AI supports:

- product understanding
- visible-text extraction
- condition interpretation
- character identification
- search-query generation
- qualitative comparable review
- explanation drafting
- reflection drafting

AI does not authoritatively determine financial results.

---

# 2. AI Responsibilities

## Product Image Analysis

Inputs:

- product images
- source title
- source category
- source price
- optional profit-pattern context

Outputs:

- category
- manufacturer
- brand
- model-number candidates
- visible text
- character
- new or used
- condition
- dirt and wear
- approximate size class
- apparent recency
- estimated original-price band
- search terms
- uncertainties
- per-field confidence

## Lightweight Triage

For products unresolved by deterministic rules, AI may output:

- likely relevant
- likely irrelevant
- uncertain
- reasons
- suggested category

The deterministic application decides whether the product advances.

## Comparable Review

AI reviews only a limited number of high-ranking comparable candidates.

It may assess:

- same product
- same series
- similar only
- incompatible
- condition similarity
- attribute mismatch
- authenticity concern

## Explanation Drafting

AI may convert structured reason components into concise Japanese.

Structured reasons remain authoritative.

## Reflection

AI may summarise:

- successful searches
- failed searches
- emerging patterns
- source problems
- suggested next themes

Reflection cannot directly alter scoring configuration.

---

# 3. AI Prohibitions

AI must not be the authoritative source for:

- Mercari fee
- resale shipping
- sourcing cost
- expected profit
- profit margin
- sales prospect score
- confidence score
- overall sourcing score
- recommendation tier
- quantity recommendation
- purchase decision

# 6. Prompt Structure

Each prompt contains:

1. role
2. task
3. permitted evidence
4. prohibited assumptions
5. output schema
6. uncertainty rules
7. examples
8. language requirement
9. prompt version

Core instruction:

```text
Do not guess unreadable model numbers.
Return multiple candidates when uncertain.
Use null for unknown values.
Do not calculate profit, fees, shipping, or recommendation scores.
```

---

# 7. Prompt Versioning

```text
prompts/
  product_analysis/
    v1.md
  comparable_review/
    v1.md
  explanation/
    v1.md
  reflection/
    v1.md
```

Each execution stores:

- prompt family
- prompt version
- schema version
- provider
- model
- request hash
- raw response
- parsed response

Prompt updates require version increment and regression evaluation.

# 9. Image Selection

Priority:

1. full product
2. manufacturer logo
3. model label
4. packaging
5. condition detail
6. accessories

Default detailed-analysis limit:

```text
5 images
```

Permitted preprocessing:

- resize
- orientation correction
- crop
- duplicate removal
- quality validation

---

# 10. OCR Strategy

Phase 1:

1. Gemini extracts visible text.
2. Text candidates and confidence are retained.
3. Model candidates generate staged searches.
4. Material ambiguity triggers user confirmation.
5. A dedicated OCR adapter may be added later.

Generative extraction is not treated as exact OCR.

---

# 11. Validation

Use strict typed validation such as Pydantic.

Rules:

- schema version required
- confidence 0–1
- severity 0–1
- enums validated
- unknown represented as null
- unexpected financial-decision fields rejected

On invalid output:

1. store raw output
2. optionally retry once with correction prompt
3. otherwise mark analysis unavailable
4. continue the run

---

# 12. AI Confidence

Field confidence is not recommendation confidence.

Example:

```text
manufacturer confidence = 0.98
model confidence = 0.55
condition confidence = 0.70
```

The deterministic confidence scorer combines AI and market evidence.

# 14. Explanation Architecture

Input:

- structured positive reasons
- negative reasons
- risks
- assumptions
- evidence counts
- recommendation tier

Output:

- concise Japanese
- no new facts
- no recalculation
- no confidence inflation

Example:

```text
推奨理由:
・直近3か月で状態の近い売却実績が8件あります。
・仕入れ価格が相場中央値より十分低く、予想利益は620円です。
・小型商品のため送料推定の確度が高いです。

注意:
・型番末尾が画像では判別しにくいため、購入前に確認してください。
```

# 15. Reflection Architecture

Input:

- run metrics
- source successes and failures
- candidate counts
- recommendation reasons
- rejection reasons
- user corrections
- actual outcomes when available

Output:

- observations
- hypotheses to investigate
- unproductive searches
- source issues
- suggested next focus

Prohibited:

- direct database mutation
- direct score update
- automatic threshold changes

---

# 16. Cost and Usage Control

Record:

- call count
- image count
- input and output size when available
- cost when available
- model
- latency
- failure type

Budget categories:

- detailed product analysis
- comparable review
- explanation generation
- reflection

When budget is exhausted:

- complete with available deterministic results
- mark unprocessed items
- do not fabricate analysis

---

# 17. Retry and Failure

Retry only for:

- timeout
- temporary provider error
- permitted rate-limit retry
- malformed output with plausible correction

Do not repeatedly retry:

- unsupported image
- policy refusal
- consistently invalid content
- missing image

---

# 18. Security and Privacy

Do not send unnecessary personal data.

Never send:

- application secrets
- GitHub tokens
- Supabase keys
- unrelated personal notes
- buyer or seller personal identifiers

---

# 19. Evaluation Dataset

Fixtures should contain:

- source image reference
- expected category
- expected manufacturer
- expected model candidates
- expected character
- expected condition
- known ambiguity
- expected search terms

Metrics:

- category accuracy
- manufacturer accuracy
- exact model accuracy
- model-candidate recall
- character accuracy
- condition agreement
- schema-valid response rate
- honesty about unknowns
- user correction frequency

Reliable structured extraction is more important than fluent prose.

---

# 20. AI Acceptance Criteria

1. Product analysis returns validated structured JSON.
2. Unknown values remain null or candidates.
3. Prompt, model, and schema versions are stored.
4. AI never supplies final financial calculations.
5. Invalid output does not crash a run.
6. Comparable review is limited to top candidates.
7. Explanations use stored reasons only.
8. Provider replacement is possible.
9. Usage is logged.
10. Tests use fixtures rather than live calls.
